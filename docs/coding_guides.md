# コードガイドラインとアーキテクチャ

## アーキテクチャ
本ゲームはシンプルなMVC（モデル・ビュー・コントローラ）パターンを採用しています。

- **Model**：ゲームデータ（Player、Enemyなど）を表し、ターンの進行を管理します。
- **View**：UI要素やアニメーションの表示を担当します。
- **Controller**：ユーザー入力を受け取り、ModelとViewの連携を調整します。

ゲームプレイの中心となる`BattleManager`スクリプトがターンの管理、アクションの実行、ダメージ計算、UI更新を行います。

## コーディング規約
- C#の慣習に従い、クラスやメソッド名はパスカルケース、ローカル変数はキャメルケースで記述します。
- マジックナンバーの使用を避け、定数はクラスの先頭で宣言します。
- メソッドは単一の責務に絞って短く保ちます。
- システム間の疎結合を保つため、UnityEventやC#のイベントを活用します。

## エラーハンドリング
- UIからの入力は処理前にバリデーションを行います。
- ファイルI/O（例：保存/読み込み）の周囲にはtry/catchを使用します。
- 例外が発生した場合は、UnityEngine.Debugを使用して十分なコンテキストと共にログを出力します。
